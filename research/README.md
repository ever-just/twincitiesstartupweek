## Project Overview

### Objective
Conduct comprehensive OSINT research on Twin Cities Startup Week to compile all available historical data, sponsor information, event details, and related documentation.

### Scope
- **Time Period:** All available years (2015-2025)
- **Data Types:** Events, sponsors, speakers, attendees, media, documents
- **Geographic Focus:** Minneapolis/St. Paul, Minnesota
- **Industry Focus:** Startups, venture capital, entrepreneurship

### Expected Outcomes
- Complete sponsor database (all years, all tiers)
- Complete speaker database (all sessions, all years)
- 50+ media articles collected and summarized
- Year-by-year timeline (2015-2025)
- Investor participation database
- Startup participation database
- Comprehensive analysis and insights
- Organized folder structure
- Source documentation for all data

## Key Data to Collect

### Sponsors
- Company name, industry, website
- Years sponsored, sponsorship tier
- Sponsorship amount (if public)
- Logo and company description
- Contact information

### Speakers
- Name, title, company
- Session information
- LinkedIn profile
- Speaker bio
- Video recordings (if available)

### Events
- Event dates and duration
- Venue(s) and locations
- Event theme
- Number of sessions
- Estimated attendance
- Key announcements

### Media Coverage
- Publication, title, author, date
- URL and full text
- Key topics and quotes
- Sponsors/speakers mentioned
- Reach and sentiment

### Investors & Startups
- Investor names and firms
- Startup names and founders
- Funding data
- Post-event outcomes
- Success stories

## Tips for Success

### Research Tips
- Take screenshots of important pages
- Save URLs for all sources
- Note access dates
- Document any conflicts
- Keep detailed notes
- Follow leads and connections
- Don't skip verification

### Organization Tips
- Use consistent naming conventions
- Create clear folder structure
- Use templates for data entry
- Back up data regularly
- Version control important files
- Create index/table of contents
- Document methodology

### Time Management
- Set daily goals
- Track progress
- Take breaks
- Don't rush verification
- Batch similar tasks
- Use templates to save time
- Automate where possible

## Troubleshooting

### Problem: Behind Schedule
**Solution:** Extend timeline, focus on high-priority data first, use AI agents to accelerate

### Problem: Data Not Found
**Solution:** Try alternative search terms, check related sources, use Wayback Machine, note as gap

### Problem: Conflicting Information
**Solution:** Check publication dates, verify source credibility, cross-reference, document conflict

### Problem: Overwhelmed
**Solution:** Break into smaller chunks, focus on one topic at a time, use templates, take breaks
